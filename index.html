Creational Pattern

1. Object Pool

형태

객체를 미리 생성

필요할 때 가져다 쓰는 패턴

용도

객체 생성 및 초기화 비용이 클때 사용

많은 수의 인스턴스를 생성하는 경우 사용

Creational Pattern

2. Multiton Pattern

형태

싱클턴 패턴의 확장판

1개가 아닌 n개의 인스턴스를 관리

용도

로드 밸런스가 n개의 서버 중, 사용 가능한 한개의 서버를 반환하는 경우

Structural Pattern

1. Data Mapper

형태

CURD Interface 제공

e.g) create, update, read, delete

SQL을 실행

지정한 Object로 결과 전달

용도

Domain Layer와 Persistence Layer를 분리할 때 사용

e.g) MyBatis

Structural Pattern

2. Fluent Interface

형태
							
query = new Sql()
	.select(['foo', 'bar'])
	.from('foobar', 'f')
	.where('f.bar = ?');

chaining 기법으로 구현

용도

마치 사람이 사용하는 언어와 같이 Interface를 구현

Structural Pattern

3. Registry

형태

객체를 저장하고 해당 객체를 리턴 하는 Class

용도

어플리케이션 전역에서 사용하는 객체 로딩

e.g) Logger

단점

Global State를 사용하게 되는 안티패턴

DI 사용 권장
